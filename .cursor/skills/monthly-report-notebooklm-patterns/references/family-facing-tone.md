# ご家庭向け月次レポート：文体・プロンプト指針

NotebookLM の `notebook_query` や、Markdown／HTML への清書時に **第三者の観察レポートのようにならない** ための指針です。

## 避ける表現（他人事・過度に客観）

- 「〜が挙がっております」「〜と記載されています」「〜と整理です」「推奨されております」だけの**観察者口調**の連続。
- 「〜に見えます」「〜段階に見えます」など、**当塾の関与が消える**言い方。
- 生徒名・担当名を出さず「一般論」だけを並べる文。
- 「順次ご説明してまいります」など、**誰が・いつ・何を**が抜けた抽象句の多用。

## 推奨する立ち位置

- **当塾／担当教師の一人称**（「当塾では」「森田としては」「私ども」）と、**お子さまのお名前（○○さん）**を適度に織り込む。
- 事実はマップ許可ソースに限定しつつ、**見立て・ねらい・お願い**は「私たちが伴走している」文脈で書く。
- 課題は「データ上の欠陥」ではなく、**一緒に力を入れたい点**として伝える。
- 保護者への依頼は「ご協力をお願いします」より、**「こうしていただけると、授業がさらに活きます」**のように理由とセットにする。

## NotebookLM `notebook_query` に足す一文（例）

クエリの末尾に次を**必ず含める**こと。

> 出力は保護者向けの月次レポート用です。観察者や第三者のレポート調（「〜と記載があります」「〜が挙がっております」だけの連続）にせず、当塾・担当教師がお子さまに伴走している口調（です・ます）で書いてください。生徒のお名前を適度に入れ、推奨や課題は「当塾では〜したい」「担当として気にかけている点は〜」のように当事者として述べてください。ソースにないことは書かない。**配布では「担当CA」「教師MTG」「Gemini メモ」を書かず「担当」とする。計画表シート俗称は本文に並べず、リンクは「▶ 学習計画表」の短文で。01 の学校テストは略号メモを貼らず保護者向けに一文で要約する。01 の IB 目標／現状は計画表と列対応させ入れ替えない。ご家庭へのお願いに IA と EE を含める。仮の単一授業日だけはレポート本文に載せない。**構成は **`docs/samples/monthly-reports/monthly_pattern_b_content.template.md`** に準拠する。**03 授業内容**では、表の直下に「理解度（★）の目安」を次の5行を**一字一句変えず**含めること：★☆☆＝基礎内容の定着が必要／★★☆＝基礎問題が解けるが応用問題に対応できない／★★★＝基礎に加え応用問題にも対応できている／※いずれも授業内の観察と演習の様子に基づきます。別の言い換えに置き換えない。

## ご家庭の画面に載せない語（Pattern B）

次は**配布用 HTML に書かない**（社内向けドラフトや管理者脚注は別）。

- 「担当CA」→ 表・本文とも **「担当」**
- 「教師 MTG」「Gemini メモ」「NotebookLM」
- 計画表**シート俗称のみ**を本文で列挙しない（リンクの表示は短文 CTA）。
- スプレの**略号だけの試験メモ**は貼らず、Topic／unit test の平易な日本語に要約

## 構成上の鉄則（編集レビュー反映）

- **02 良い点**：**評価が明示できる結果を先頭1文**。長い単元は **03**。
- **02 mood 第3列**：その月の**観察・打合せの具体**を1つ入れる。
- **04**：ソースにあれば計算ミス対策として**マークスキーム確認**を書いてよい。創作は禁止。
- **05・07**：**仮の単一授業日のみ**は家庭向け本文に載せない。
- **複数生徒 MTG**：教師メモが複数名・複数教科を扱うときは、`docs/samples/monthly-reports/monthly_pattern_b_content.template.md` の **「複数の生徒が同一 MTG／同一 Doc に登場するとき」**に従い、特に **文頭が（対象の）○○様で始まる段落のみ** を根拠にする運用が安全。別姓＋様の段落は他生徒向けとして採らない。
- **03**：長いサブトピックは **`docs/samples/monthly-reports/monthly_pattern_b_template.html`** の `.unit-main` / `.unit-subtopics` で整形（インライン `style` の乱刷り回避）。

## 03 授業内容 — 理解度（★）の目安（固定文案 HTML／MD 共通）

**授業日・単元の表の直下**に、次を **そのまま** 載せる（意訳・略・別定義に差し替え禁止）。

| 記号 | 意味 |
|------|------|
| ★☆☆ | 基礎内容の定着が必要 |
| ★★☆ | 基礎問題が解けるが応用問題に対応できない |
| ★★★ | 基礎に加え応用問題にも対応できている |

※いずれも授業内の観察と演習の様子に基づきます。

学習計画表に「理解度」列がない場合は、その旨を **同じブロックの末尾** に一文足す（例：「学習計画表に理解度列がないため、担当の見立てで記載しています。」）。正本は **`docs/samples/monthly-reports/monthly_pattern_b_content.template.md`** の **03 授業内容** 節。

## 推敲チェック（第2回・第3回に追加してよい観点）

- [ ] 「記載があります／整理です」が残っていないか → **当事者の文に言い換えたか**
- [ ] 生徒名・担当名が十分に入っているか
- [ ] 保護者が読んだときに**冷たい調査報告**に聞こえないか
- [ ] **03（授業内容）** の表の直後に、**理解度（★）の目安**が固定文案どおり入っているか（言い換え・欠落がないか）
- [ ] **配布語彙**：担当CA／教師MTG／シート俗称の直書き／スプレ生メモの貼り付けが無いか
- [ ] **01**：IB 目標・現状の列取り違えが無いか

## 書き換え例（同一事実）

| 避けたい | 望ましい方向 |
|----------|----------------|
| 春休みに向けての課題として、トピック A の発展問題への慣れが挙がっております。基本の定着に加え、ExamMate など…推奨されております。 | 春休みに向けて、**当塾では**トピック A の発展問題に**少しずつ慣れていくこと**を大切にしたいと考えております。謙吾さんには、基本の定着に加え、ExamMate など試験に近い形式の問題にも触れていただき… |
| 計画どおり進んでいる段階に見えます。トピック C に入る前後は、今後の授業とあわせて順次ご説明してまいります。 | 学習計画表では、トピック A・B の多くで学校・当塾とも「済」が入っており、**謙吾さんのペースは計画とよく合っている**ように思います。トピック C に入るころは、**森田の授業の中で**理解の度合いに合わせて、改めて丁寧にご説明いたします。 |
