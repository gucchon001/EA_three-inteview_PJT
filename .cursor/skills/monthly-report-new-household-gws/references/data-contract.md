# データ契約（「2 シート」+ MTG）

ここでの **「学習計画表の 2 シート」** は運用上の最小セットとして次を指す。

## 必須ブロック

| シート（既定の名前） | Pattern B で主に効く節 | 備考 |
|---------------------|-------------------------|------|
| **`student`** | **01 基本情報**（mock/final、志望、科目一覧など） | gid はブックごとに異なる。`spreadsheets.get` で確認。 |
| **`【SL】lesson plan`**（またはコースが SL でない場合の **同等の「進行表」タブ**） | **03 授業内容**、**05**、**07** の表・日程・単元 | 十倉ブックでは gid `116668180`。名前が違うブックではメタで `title` を確認し、`fetch_monthly_gws_sources.py` の range オプションで合わせる。 |

## HL を履修している場合の追加

| シート | 用途 |
|--------|------|
| **`lesson plan`**（ブックによっては非表示） | HL 科目側の進行・学校進捗がこちらのみのとき、**HL 科目の記述**や表に反映 |

Math のみ SL で HL が別紙のときは、**レポート本文で HL に触れるか**は編集方針で決める（触れるならこのタブも取得）。

## 教師ミーティング（Google Doc）

| 入力 | 用途 |
|------|------|
| **文字起こし・Gemini メモ等の Doc** | **02 塾での様子**、**04 課題とアドバイス**、**07** の文章・方針・家庭へのお願い |

API 取得は JSON のままでは編集しにくいため、**プレーンテキスト抽出**（`gws_doc_json_to_plaintext.py`）を推奨。

## セクション対応早見（詳細は規約テンプレ）

| 節 | 主ソース |
|----|----------|
| 01 | `student` |
| 02 | MTG（mood は **学習進捗・学習意欲・宿題の提出** の3行） |
| 03 | lesson plan 表 + MTG の所見 |
| 04 | MTG |
| 05 / 07 | lesson plan + MTG（重複は削る） |
