## 2. ポータル IA（サイドバー + グローバルメニュー）

**ベンチマーク（スキル）**: 入口の情報優先順は **`dashboard-first-view-ux`**（「自分宛て・アラートを最上段」／Jira 型の要約・担当リストの考え方を参照）。ナビの可読性・コントラストは **`ui-frontend-patterns`**。視覚トークンは **[DESIGN.md](../../DESIGN.md)**（左アクティブボーダー等）。

**実装（モック）**: `base.html` / `portal_frame.py`。**Cool utilitarian**（`DESIGN.md` §1）— ティールの髪線シャドウ・ブランド円点・`:focus-visible`・44px ナビ行。

### 2.1 グローバルメニュー（上部・常時固定）

| 要素 | 役割 |
|------|------|
| **プロダクト名** / ロゴ | クリックで `/`（ロール別ダッシュ）へ |
| **ロールバッジ** | 管理者 / 教師 / 経理（誤操作・権限の文脈提示） |
| **並行運用** | [FR-8.3](../requirements/v0.3.md): `parallel_operation` が on かつ当該データに URL があるときだけ **「元のシートを開く」**（外部リンクアイコンで明示） |
| **自分宛て** | 常時 **`/me/todos` へのショートリンク**（バッジ件数は API 連携後） |
| **ユーザーメニュー** | 表示名・ログアウト（将来: プロフィール） |

※全文検索・通知のベルは Phase 2 以降でよい（当面はダッシュ＋ ToDo で代替）。

### 2.2 サイドバー（左・ロール別）

**単一の情報構造**で迷いを減らし、**教師には管理系を非表示**（サーバ側でメニュー項目を生成）。

| グループ | 項目 | 主ロール |
|----------|------|----------|
| **ホーム** | ダッシュボード `/` | 全員 |
| **マスタ** | 教師一覧 `/teachers` | 管理者（教師には非表示） |
| | 生徒一覧 `/students` | 管理者・教師 |
| **運用** | 月次レポート一覧 `/reports/monthly` | 管理者（教師は自分担当への導線はダッシュから） |
| | 面談・記録（新規はコンテキストリンク中心） | 管理者・教師 |
| | 自分宛 ToDo `/me/todos` | 管理者・教師 |
| **課金・契約** | 契約書・口座まわり（教師詳細へ誘導） | 経理・管理者（Phase 1 は詳細画面からで可） |
| **システム** | 設定 `/settings` | 管理者のみ |
| | 取込エラー `/import-errors` | 管理者のみ |

**操作感**: アクティブ項目は [DESIGN.md](../../DESIGN.md) の左ボーダー 3px。**グループ見出し**は小さく muted（スクロール長大化を防ぐ）。幅は 220〜240px 固定、**タブレット以下**はアイコンのみに縮小 or ハンバーガーで開閉（`prefers-reduced-motion` 考慮）。

### 2.3 パンくず・メイン領域

- **パンくず**: グローバル直下。`ホーム > 生徒一覧 > 山田太郎` のように **1 画面＝1 主目的**で浅く保つ。
- **メイン**: `max-width: 1280px`、一覧は横スクロール許容。**ページ単位の主要 CTA**（例: 「生徒を追加」）はタイトル行右寄せ（`dashboard-first-view-ux` の「アクション可能を前へ」）。

### 2.4 ダッシュボードとの役割分担

- **ダッシュ**: 「いま何に手を付けるか」の**例外・期限・自分宛て**（スキル推奨の並び）。
- **サイドバー**: **全機能への安定した入口**（ブックマーク不要で辿れる）。
- **グローバル**: **自分宛て・シート退避・ユーザ**, **アプリ横断の最短動線**。

## 4. 画面別メモ（追記用）

### SCR-D01 / SCR-D02 ダッシュボード

| ブロック（上→下） | 管理者 | 教師 |
|-------------------|--------|------|
| 1 | 未提出月次／充足率異常／面談関連のサマリー | **自分宛て・今月締切・ToDo** |
| 2 | 直近アラート一覧（HTMX 断片可） | 担当スロット一覧への導線 |
| 3 | ショートカット（教師・生徒・レポート） | 同上 |

### SCR-T01 教師一覧

- 列: v0.3 F-1.2 を参照。フィルタ: アクティブのみ既定、検索（氏名・メール）。
- 行アクション: 詳細、招待再送（`invited` 時）。

### SCR-S01 生徒一覧

- 教師ロール時は **自担当生徒のみ**（サーバ側フィルタ）。

### SCR-A01 指導枠詳細

- 上部: 選考番号・科目・契約時間・教師。下部: **月次タイムライン**（年月 × 実施・充足率・提出）。
- データソース表示: 契約時間はFileMaker/基幹情報、実施時間・指導報告書はマイページ（指導報告書）BigQuery、面談/MTGステータスは移行期のみ指導モニタリング管理表を参照していることを、管理者向けに確認できるようにする。
- 差分/アラート: BigQueryの指導報告書実績と学習計画表/ポータル計画が乖離した場合、該当月のタイムライン上に要確認バッジを表示する。

### SCR-S02 生徒詳細

- 基本情報は将来的にstudentシートではなくポータルDBを正とする。移行期は元Sheetsリンクを残し、学習計画表作成MVPの組み込み時点でstudentシート運用を廃止する。
- 生徒詳細から、指導枠、学習計画表、面談/教師MTG履歴、レポート工房の生成履歴へ遷移できる構成にする。

### SCR-C01 設定

- 鍵: `monthly_report_deadline`, `fulfillment_rate_warn`, `reminder_days_before`, `parallel_operation` 等。変更は `audit_log`（FR-9.3）。
