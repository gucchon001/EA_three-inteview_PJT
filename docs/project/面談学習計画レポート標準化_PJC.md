## why

### 背景（AsIs / ToBe / ギャップ）

**【現状（AsIs）】**  
面談メモが属人的で、準備に **4〜6 時間／件**かかる。教師ミーティング約 60 分・家庭面談 30〜60 分に加え、前後準備が重い。開発は面談メモビューと MOOC 中心。教材・教師連携は停滞し優先度低下。スケジュールはテスト時期・担当変更・順番変更などイレギュラーが多く、システム化と手動の線引きが未決。顧客向けコミュニケーションはメール中心で、**アップセルにはレポート（PDF 等）の方が有効**という認識。模試・答案・日程などの情報共有が遅延・未共有になりやすい。週次指導報告は手作業が多く、月次サイン・給与連携とサイクルが必ずしも一致しない。教師ミーティング内容のメール化が上期の最大工数（4〜6 時間／件）で、AI 要約の品質が低く藤井さんによる大幅修正が必要な状態が継続している。

**【あるべき姿（ToBe）】**  
ビューを **「面談メモ」**に集約し、編集・出力を簡便化。テンプレを複数（学校ベース・サブトピック数ベース等）用意し、選択後にカスタマイズ。**ブラックボックス化を避け構造を可視化**。成果物は **メール中心からレポート中心**へ。教師ミーティング起点の **月次ルーチン**（教師 MTG → メール／家庭面談※順序は顧客都合で変動）で **アップセルと離脱防止（接触回数確保）**を両立。教師ミーティングは **30 分・メール作成も 30 分**で完了できる業務設計を目指す。週次指導報告は録画文字起こしをそのまま自動送付する**全自動化**（承認プロセス省略・請求書が事実上の承認）とし、アカデミーの品質保証はレポート・面談で担保する。月次報告はメール報告に統合できるかを検討する。

**【ギャップ・原因の仮説】**  
ツールとテンプレが分散し標準化されていないこと、要約品質の定義が言語化されていないこと、入力データの十分性が未検証なことが工数と品質のボトルネック。スケジュール変動はルール化より先に「基本枠組み」と例外処理の設計が必要。

## 目的・成功状態

### 【プロジェクトの 3 本柱】

1. **アップセル促進** — レポート・面談の接触回数と質を高め、追加契約につなげる  
2. **離脱防止** — 月次ルーチンで定期的な接触を確保し、契約継続率を上げる  
3. **業務標準化** — 一般職・アルバイトでも対応できるよう、型化・自動化で属人性を排除する

### 【卒業要件】（親PJC全体を完了とみなす条件）

1. **月次レポート制作パイプライン**が実運用で回っていること（P1a〜P1d完了）。文字起こし自動取り込み→AI生成→藤井さん編集→ワンタイムURL配信の一連が10件/月で安定稼働。
2. **週次指導報告の自動化**（P3）について、設計叩き台とパイロット範囲が合意され、実証が始まっていること。
3. **品質基準**が言語化され、AI生成レポートの修正率が20%以下で安定していること。
4. **未決事項一覧**（法務・権限・SLA・承認フロー等）がオーナー付きで棚卸しされ、次四半期に持ち越す項目が明示されていること。

### 施策詳細

1. **P1a テンプレ決定・手動パイロット**: 4パターン（A〜D）から採用テンプレを確定。藤井さんの直近面談1件の実データで手動レポートを作成し、保護者に配信。フィードバック収集。**パイロット配信は簡易方式（Drive限定共有 or 簡易パスワードページ）で行い、ワンタイムURL認証はP1bで本格開発する。**
2. **P1b AI生成＋品質検証 / セキュリティ開発**: Gemini 単体で3ソース（教師MTG記録・家庭面談記録・学習計画表）からレポートを自動生成。品質チェックリストを策定し、藤井さんの修正率20%以下を目標に検証。並行してワンタイムURL認証バックエンド・保護者向け閲覧ページ・DB設計を開発。File Search（Gemini）のストア設計は準備のみ行い、2ヶ月目以降の時系列比較で本格導入。
3. **P1c マイページ開発**: 社内管理画面（レポート一覧・編集UI・承認→送信フロー）＋文字起こし手動アップロード＋ご家庭ビュー（過去レポート閲覧）を開発。藤井さんが画面上で編集〜送信を完結できる状態にする。
4. **P1d 配信自動化＋取り込み自動化**: メール自動送信（SendGrid等）・送信履歴・開封確認の配信基盤。Google Drive からの文字起こしファイル自動取り込みパイプライン。命名規則統一・教師発言ルール導入。10件以上でも破綻しない仕組み。
5. **P3 週次指導報告の自動化**: P1b の生成エンジンを流用し、録画→文字起こし→報告書→共有を全自動化（承認プロセス省略・請求書が事実上の承認）。
6. **P4 無料見積もり診断サービス**: P1/P3 のデータ蓄積を活用し、答案提供家庭への AI 分析レポート無料提供。後段で着手。
7. **スコープ外（当面）**: 三者面談の新規プロセス立ち上げ、教材の同時大型推進。

### AI 技術方針

| 段階 | 方式 | 理由 |
|------|------|------|
| P1b 初期 | Gemini 単体 | 3ソースをプロンプトに直接渡して品質検証。最速で試せる |
| P1b 後期〜P1c | Gemini File Search 導入 | 2ヶ月目以降で「先月比」が必要。生徒ごとのストアに蓄積 |
| P1d〜 | File Search 本格運用 | ストア自動管理・自動取り込みと連動 |
