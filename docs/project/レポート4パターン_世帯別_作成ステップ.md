# ご家庭ごとに Pattern A〜D を作る手順（ステップバイステップ）

## 前提

- NotebookLM ノート **EA_生徒状況分析**（`notebook_id` は `household_registry.json` 参照）に、対象世帯の **260319 時系列まとめ** ソースがあること。
- CLI **`nlm`** が利用でき、`PYTHONUTF8=1` で `nlm content source` が本文を書き出せること。

## ステップ 1 — 世帯をレジストリに登録

`samples/reports/household_registry.json` の `households` に、次を定義する。

- `slug` … フォルダ名 `household_<slug>`
- `family_house` … レポート見出し用（例: 十倉家）
- `pattern_honorific` … 十倉さま（とくらさん）のように **呼称＋よみ**
- `meta_student_line` … メタ情報欄の一行
- `student_id` … 受講番号（未確定は `TBD`）
- `course` … コース表記（タイトル行に使う）
- `timeline_source_id` … 260319 時系列ソースの UUID
- `nl_drive_blurb` … NotebookLM 説明文用の短い表記

## ステップ 2 — 一括生成

プロジェクトルートで:

```bash
set PYTHONUTF8=1
python scripts/generate_household_patterns.py
```

単一世帯だけやり直す場合:

```bash
python scripts/generate_household_patterns.py --slug takafuji
```

生成物:

- `samples/reports/household_<slug>/pattern_a_responsive.html` 〜 `pattern_d.html`
- `sources/00_260319_時系列まとめ_NotebookLM.md`（NotebookLM から取得）
- `sources/_nl_timeline.json`（再現用）
- `sources/03_学習計画表_プレースホルダー.md` / `04_参照メール_プレースホルダー.md`
- `sources/notebooklm_source_ids_<slug>.md` / `README.md`

## ステップ 3 — 人手レビュー（必須）

自動生成は **household_giga（儀賀）をテンプレに置換**したものです。次を必ず確認する。

1. **数値・KPI** … 儀賀のデモ値は `—` またはゼロに落としてある。**03 学習計画表のスナップショット**を `03_学習計画表_YYYYMMDD.md` として差し替え、本文・グラフ・表を更新する。
2. **01・02 相当** … NotebookLM の教師MTG・ご家庭提案を `source_get_content` で取り、`sources/` に保存（ファイル名は世帯で統一し、HTML 内リンクを合わせる）。
3. **ファクトチェック表** … `#factcheck` の照合内容を、その世帯のファイル名に合わせて直す。
4. **小林 慧さま** … `student_id` が `TBD` のため、Drive／計画表で番号を確定し、レジストリと HTML を更新する。

## ステップ 4 — 一覧の確認

`samples/reports/index.html` に行を追加・更新し、`python scripts/serve_project.py` でリンクと文字化けを確認する。
