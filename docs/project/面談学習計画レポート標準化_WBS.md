## 体制

| 役割 | 担当 | 責務 |
|------|------|------|
| 開発主体・PM | 原口 | システム開発、AI連携、テンプレ、WBS管理 |
| 面談実施・品質確認 | 藤井 | 教師MTG・家庭面談の実施、レポート品質レビュー・修正 |
| 現場責任者 | 元山 | 業務フロー決定、運用ルール承認、教師マネジメント |

## フェーズ構成

```
P1a テンプレ決定・手動パイロット ← 今ここ
P1b AI生成＋品質検証 / セキュリティ開発（並行）
P1c マイページ開発（社内管理画面＋ご家庭ビュー）
P1d 配信自動化＋文字起こし自動取り込み
P3  週次指導報告の自動化（P1b エンジン流用）
P4  無料見積もり診断サービス（後段）
横串 ガバナンス（品質定義・未決クローズ・法務）
```

## 肝タスク

| 概要 | 理由 |
|------|------|
| テンプレパターン決定 → 実データ1件パイロット | ご家庭に最速で届けてフィードバックを得る |
| 要約品質の定義・評価指標の合意 | 「ほぼそのまま送れる」AI品質の判定基準がないと P1b が進まない |
| セキュリティ配信基盤（ワンタイムURL） | レポートを安全に届ける仕組みが全フェーズの前提 |
| 文字起こし命名規則の統一 | 自動取り込み（P1d）の前提条件 |
| 法務・権限（録画・文字起こし・個人情報） | 未確定のままだと本番展開できない |

## フェーズ依存関係

```
P1a テンプレ決定・手動パイロット
 │
 ├──→ P1b AI生成＋品質検証 ←──── 横串: 品質定義
 │     ‖ 並行
 │     P1b セキュリティ開発
 │     P1b DB設計
 │     P1b File Search設計（準備のみ）
 │
 ├──→ P1c マイページ開発
 │      ├ 社内管理画面（編集・承認・送信）
 │      ├ 文字起こし手動アップロード
 │      ├ ご家庭ビュー（過去レポート閲覧）
 │      └ File Search 本格導入（時系列比較）
 │
 ├──→ P1d 配信自動化＋取り込み自動化
 │
 └──→ P3 週次指導報告（P1bエンジン流用）

P4 診断サービス（P1/P3のデータ蓄積後）
```

## AI技術方針

| 段階 | 方式 | 理由 |
|------|------|------|
| P1b 初期 | Gemini 単体 | 3ソースをプロンプトに直接渡して品質検証。最速で試せる |
| P1b 後期〜P1c | File Search 導入 | 2ヶ月目以降のレポートで「先月比」が必要。ストアに蓄積 |
| P1d〜 | File Search 本格運用 | 生徒ごとのストア自動管理・自動取り込みと連動 |
