# Phase 0｜PO レビュー用パッケージ

> 原口様（PO）へ Phase 0 で一度に確認いただく項目の統合表。  
> 出典: [要件定義 v0.3 §9](../requirements/v0.3.md)（C-1〜C-10）、[パーサー仕様 v0.1 §10](../sheets-migration/parser-spec.md)（1〜6）

**期待する返答形式**: 各項目に「決定内容」または「要検討・別途打ち合わせ」を追記いただくか、口頭／メールで回答をいただければ、開発側で v0.4 および parser-spec の REV 欄に反映する。

---

## 1. 重複を統合した論点一覧

| ID | テーマ | 要点 | 出典 |
|----|--------|------|------|
| **P-1** | 苗字のみの教師名が複数 canonical に該当 | 手動マッピング UI の要否、その他の運用 | C-5、parser §10-1 |
| **P-2** | `Math(HL)` 等、レベル・種別が不明な科目表記 | 正規化ルール（例: HL 既定＋マスタ拡張、unknown バケット等） | C-6、parser §10-2 |
| **P-3** | Sheets 上の独自列の取り込み範囲 | 事業上、DB に残すべき独自列の有無 | C-3、parser §10-3 |
| **P-4** | 退職教師が canonical（教師マスタ）にない場合 | マスタ復活登録 vs 別キーで履歴保持 等 | C-7、parser §10-4 |
| **P-5** | 並行運用中の自動インポート頻度 | 毎晩 / 週次、その他（手動のみは非推奨だが選択肢には含めない想定） | C-8、parser §10-5 |
| **P-6** | dryrun の差分警告閾値 | 例: `monthly_reports` N 件以上更新で警告。N の既定値 | C-9、parser §10-6 |
| **P-7** | 招待メール | 文面ドラフト・差出人ドメイン（From） | C-1 のみ |
| **P-8** | 下部記録表の正本 | 2 形式（原本 vs 三者面談・教師mtg）のどちらを単一ソースとするか | C-2 のみ |
| **P-9** | 仕様レビュー体制 | 参加メンバー確定（原口・元山・藤井＋追加の有無） | C-4 のみ |
| **P-10** | 設定画面の権限分離 | 締切・閾値変更 vs 並行運用フラグ等、ロール分割の要不要 | C-10 のみ（v0.3 では「保留（後日）」） |
