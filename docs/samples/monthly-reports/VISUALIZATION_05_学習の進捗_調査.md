# ⑤学習の進捗：トピック別「学校 vs 塾」ビジュアライズ可否調査

**結論**: **実装可能**です。月次レポートが **静的 HTML + 印刷/PDF** 前提でも問題ありません。難所は **データを「10%刻み（または10マス）」でどう定義するか**（学習計画表の「済」だけから%を推定するか、手入力するか）です。

**参照画像**（ユーザー提供）: [assets/review_layout_05_progress_bars_reference.png](assets/review_layout_05_progress_bars_reference.png)

---

## 1. 要件の整理（添付イメージ）

| 要素 | 内容 |
|------|------|
| 行 | トピック 1〜n。各トピックに **学校** 行と **塾** 行 |
| 列 | 0〜100% を **10 マス**（各 10%）で表現 |
| 色 | **青**＝済、**グレー**＝未、**黄**＝直近1ヵ月の予定 |
| 目的 | トピックごとに **学校と塾の進捗を並べて比較** |

---

## 2. 実装方式の比較

| 方式 | 可否 | メリット | 注意 |
|------|------|----------|------|
| **HTML + CSS（10列グリッド／flex でセル着色）** | ◎ | Chart.js 不要。印刷・PDF に強い。Vercel 静的配信と相性が良い | マス数分の DOM が増える → **テンプレ＋スクリプト生成**または **短いループ（ビルド時）**が現実的 |
| **1行を `linear-gradient` で3色帯** | ◎ | DOM が軽い（1要素1バー）。% を CSS 変数に渡すだけ | 細かい「マス目」表現はグリッドの方が添付に近い |
| **SVG（`<rect>` を積み上げ）** | ◎ | 座標が明確。印刷も安定 | 生成ロジックは JS かビルドステップ向き |
| **Chart.js 等（横向き積み上げ棒）** | △ | 実装例は多い | バンドル増、**印刷プレビューでの解像度／凡例**の確認が必要。オフライン単体 HTML ではやや重い |
| **スプレのスクショ貼り** | ◎ | 最短 | **再現性・差し替え・自動化**が弱い（運用で許容なら可） |

**推奨（月次レポート向け）**: **CSS 10マス + 3状態クラス**（または `linear-gradient` 1行）。依存追加なしで **添付に最も近い見た目**を出しやすいです。

---

## 3. データ面の課題（ここが「できる／できない」の分かれ目）

学習計画表が **「済／未」や TRUE/FALSE だけ**の場合:

- **済 → 100% 青**のように割り切るのは簡単だが、添付の **「70% 済 + 30% 黄」** のような中間表現は **一次データに無い**。
- 対策の例:
  1. **サブトピック数ベース**: 同一トピック内の行数のうち「済」の割合を % に換算（要集計ロジック）。
  2. **手入力列**: スプレに「学校進捗%」「塾進捗%」を持たせる（運用コストは下がる）。
  3. **三値のみ**: 各マスを「済／予定／未」のどれかに離散化（% は使わず 10 マスに状態を割当）。

**「黄＝直近1ヵ月の予定」** は、学習計画表の **授業日・復習予定メモ** とレポート対象月を突き合わせる **日付ルール**が必要です。自動化する場合の **推奨既定** は [DATA_CONTRACT_05_学習の進捗.md](DATA_CONTRACT_05_学習の進捗.md) の **案 B**（対象月 `M` の暦月のみを黄のウィンドウとする。安定・見やすさ優先）。

## 4. 最小サンプル（概念：10マス × 3色）

```html
<style>
  .prog-row { display: grid; grid-template-columns: 120px repeat(10, 1fr); gap: 2px; align-items: center; margin-bottom: 4px; }
  .prog-label { font-size: 9pt; }
  .cell { height: 14px; border-radius: 1px; }
  .cell.done { background: #2e7dd4; }
  .cell.planned { background: #f5d547; }
  .cell.pending { background: #c8ccd2; }
</style>
<div class="prog-row">
  <span class="prog-label">Topic 1 · 学校</span>
  <span class="cell done"></span><span class="cell done"></span><span class="cell done"></span><span class="cell done"></span><span class="cell done"></span>
  <span class="cell done"></span><span class="cell done"></span><span class="cell done"></span><span class="cell done"></span><span class="cell done"></span>
</div>
```

実際は **学校1行・塾1行** をトピックごとに繰り返し、各マスに `done` / `planned` / `pending` を割り当てます（% から 10 マスへ量子化）。

## 5. 総合判断

| 観点 | 判断 |
|------|------|
| ブラウザ表示 | **可能** |
| 印刷/PDF | **可能**（既存レポートの `@media print` と併用） |
| 自動データ連携 | **条件付き可能**（スプレに % またはサブトピック粒度があればロジック化しやすい。「済」のみだと割合は**運用ルール**が必要） |
| 添付とピクセル一致 | デザイン調整の工数はあるが **技術的制約ではない** |
