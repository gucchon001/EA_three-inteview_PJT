# 月次レポート（対象期間・生徒名はフロントマターと冒頭表で記載）

> 本ファイルは **月次マップ準拠**の正本ドラフト。配布用 HTML には、運用に応じて本メタブロックや「推敲ログ」を載せないこと。

---

## 正本の順位（一般化）

エージェント指示・チャットプロンプトで **特定生徒ファイル名や「〇〇さんテンプレ」だけを正として差分だけ指示する** と、名前を変えるたびに再現できなくなる。**正規のソースオブトゥルース**は次の優先順位とする。

| 順位 | アーティファクト | 役割 |
|:---:|-----------|------|
| 1 | **本ファイル**（`monthly_pattern_b_content.template.md`） | セクションの意味・禁止事項・家庭向け表記・PDF レビュー由来のチェックリストの帰結先 |
| 2 | **`monthly_pattern_b_template.html`** | 見出し番号・クラス名・レイアウト骨格・主要 CSS の参照ひな型 |
| 3 | **`DATA_CONTRACT_05_学習の進捗.md`** および各 `_sources.md` でのマップ明示 | 「どのシートの列がどの `##` に行くか」のデータ契約 |
| 4（任意参照） | `monthly_YYYY-MM_<identifier>_report.html` / `.md` | **適合済みの事例・レビューで固められた配布物**。見た目確認・運用確認に使う。**規約との矛盾がある場合は規約（1〜3）側を優先し、事例側を追随させる**。 |

レビューの一次記録としての PDF 議事・アーカイブ（例：`REVIEW_*_PDFコメント反映_*.md`）は歴史ログとして読む。**配布規約への反映済み項目はテンプレ（1〜2）に読み替え済みとして扱い**、「昔のレビューをプロンプトに毎回落とす」を避ける。

### ご家庭向け文体（NotebookLM クエリ・推敲に必須）

第三者の調査レポートのようにならないこと。**「〜と記載があります」「〜が挙がっております」「推奨されております」「〜に見えます」だけの連続は禁止**。当塾・担当教師が伴走している口調（です・ます）で、生徒名を適度に織り込む。

詳細な禁止例・`notebook_query` に足す一文・書き換え例は、プロジェクトスキル **`.cursor/skills/monthly-report-notebooklm-patterns/references/family-facing-tone.md`** を参照する（**03 授業内容・理解度（★）の固定文案**も同ファイルに記載）。

### ご家庭配布での表記ルール・禁止語（編集レビュー反映）

- **語彙**：配布面では「**担当CA**」「教師 MTG」を書かず、担当者は **「担当」** とする。挨拶は「学習計画表と**担当との**打合せ」など。**本文中に計画表のシート俗称（「【SL】lesson plan」等）を並べない**。（リンクや管理者向け注記は別。）
- **01 の学校テスト**：スプレの生メモ（略号だけのメモなど）を**そのまま貼らない**。**Topic …／単元テスト で … を取得しました**など、保護者向けの一文へ要約。**見出しに「（参考）」は付けない**（運用で付けていた場合は外す）。
- **01 の IB 行**：`**IBスコア（目標）**` と `**現状**` は必ず計画表の見出しと対応させる。**両列が偶然同じ数値になるときだけ**でも、転記ミスではないか人間確認。
- **02 良い点**：**当塾で点数・評価が取れる成果を先に最大1文**。続けて総括。**長い単元説明は 03 に任せる**。
- **02 mood 第3列**：一般論だけにせず、**その月の観察・MTG で言及された具体**を1つ入れる（意欲・宿題それぞれ）。
- **03**：複数サブトピックは **公式英語**。01 の Topic 話と役割分担（01＝学校表現／03＝シラバス英語）。
- **04**：計算ミスの改善は「途中式検算」に加え、**自己採点のうえマークスキームで誤り箇所を特定する**旨を書いてよい（ソースに無いときは創作しない）。

### 文字起こし・学習計画表から生成するとき（モデルへのコピペ用）

入力が **教師 MTG の文字起こし** と **学習計画表** でも、**03 授業内容**の構成は変えない。表の直後に次を **原文どおり** 入れる（語を足す・変える・要約しない）。

```text
理解度（★）の目安：
★☆☆＝基礎内容の定着が必要
★★☆＝基礎問題が解けるが応用問題に対応できない
★★★＝基礎に加え応用問題にも対応できている
※いずれも授業内の観察と演習の様子に基づきます。
```

- 各行の ★／☆ は **計画表の列**または **MTG の所見**に基づき埋める。列がない場合は上記 ※ の直後に「学習計画表に理解度列がないため、担当の見立てで記載」など **一言**足してよい（鈴木サンプルと同様）。
- HTML に落とすときは [monthly_pattern_b_template.html](monthly_pattern_b_template.html) の **番号 03・授業内容**直下の `<p class="section-body" …>` と同じ構造・文言に揃える。

### 複数の生徒が同一 MTG／同一 Doc に登場するとき

教師による **共通のミーティングメモ／文字起こし**に、複数の生徒の様子や別教科の論点だけが並ぶことがある（Economics の担当が複数名を同席で扱う等）。その場合は次を守る。

1. **対象の確定**：まず **`_gws_student*`（学習計画表の student シート由来）にある「生徒氏名」など**で、レポート対象である生徒を一つに定める。その氏名または明らかな同一人を指す表記がない議論・数値・予定は、このレポートの**事実根拠にしない**。
2. **段落頭の ○○様で切り分ける（混在 MTG で強く推奨）**：生徒氏名から保護者向け敬称の軸になる表記を読む（多くは **姓＋様**。例:「平林　礼」→ **平林様**。メモにより **広林様** 等の表記ゆれがあり得る）。教師 MTG の **段落または箇条書き項目**について、**行頭（先頭の `[氏名]`・番号・記号は除いてよい）が、対象生徒の ○○様で始まるブロック**に含まれる文だけを、**他生徒についての事実**として採る。**別の姓名＋様**（例: **飯村様**）で始まる段落・項目の中身は、対象外生徒の事実として**すべて根拠に含めない**（誤転記しない）。  
   文頭に氏名がない段落は、**すぐ直前の段落が対象 ○○様の続き**であり、かつ内容が **本レポートの教科と対象生徒のみ** に言及すると明読できる場合に限り続きとして使う。それ以外は省略する。
3. **教科のスコープ**：レポートが **特定教科**（例: Economics SL／Physics SL）であるとき、その教科の計画・進捗に踏み込まない文（他教科だけの問題、別生徒の IA だけ等）は**転記しない**。「全体の総括」のように複数名にまたがり同定できない段落は **省略**する（創作や推測で足さない）。
4. **許容される一般情報**：複数クラス共通の運用説明でも、レポート対象である生徒の **その教科の進行**につながると明読できるときに限り、短く使ってよい。他生徒の実名・成績・提出物だけが主題の文は使わない。
5. **`--bundle-scope-reminder`（自動生成 CLI）**：科目・塾内実験用の注記を渡すときは、モデル入力の「対象レポートのスコープ」に読ませ、この節と**両方で**優先する。

### PDF・家庭レビュー準拠（プロンプト用・v3.3 系）

生成・推敲時は次を **デフォルト** とする（詳細表は [REVIEW_高藤_PDFコメント反映_v3.3.md](REVIEW_高藤_PDFコメント反映_v3.3.md)）。

1. **採番（配布 HTML）**: `sec-num` は **01 → 02（塾での様子）→ 03（授業内容）→ 04 → 05** を基本とし、**学習の進捗ブロック**があるテンプレでは **05＝比較表**、**07＝今後の授業計画**を足す。**06 は出さない**。**03a / 03b の表記は使わない**（旧資料の読み替え用語のみ）。
2. **02 塾での様子**: mood 表は **3 観点**（**学習進捗**・学習意欲・宿題の提出）。詳細な進捗比較は **05**、単元の事実は **03**／良い点ボックス。
3. **学校の試験の独立セクション**: 01 で mock/final・学校テスト状況を出しているなら **省略**（MD の `## 学校の試験（省略可）` ごと無し）。
4. **04**: **計算ミス・時期の振り返り**をここへ。計算プロセスのアドバイスは、ソースにあれば **自己採点後にマークスキームで誤った箇所を特定する**旨を優先してもよい。学校の**単元テスト**に関する長文があれば出所確認のうえ本文へ。
5. **05 / 07**: 比較表と「今後の計画」の役割分担（重複削除）。**レイアウト別名「05のみ（計画集約）」**として、進捗比較セクションを持たず **05 の番号だけで「今後の授業計画」まで担う配布構成**も可（具体例：`monthly_2026-04_takafuji_report.html` の rev v3.5.7 系）。
6. **07（ある場合）**: 案の分割をやめ **一括の本文**＋**ご家庭へのお願い**。学習計画表 URL は運用差し替え前提の注記でよい。
7. **03 授業内容**: 表の**直下**に **理解度（★）の目安**を [§03 固定文案](#03-授業内容) と **同一の語句**で必ず入れる（別表現に置き換えない）。
8. **未確定は承認前に人間確認**: 対象期間と計画表の最終正、公開 URL の可否、**二項分布等の事実**は答案・学校情報で照合。
