## ステップバイステップ検証手順

### Step 1 — スコープを固定する

1. 対象期間を **2026年4月** に限定する（レポートの「対象期間」と一致させる）。  
2. 照合先 HTML を開く: `docs/samples/monthly-reports/monthly_2026-04_tokura_v2_report.html`。  
3. セクション **01〜05** ごとに「事実」「所見」「家庭向け約束」を分けてメモする（一次ソースにあるのは主に「事実」）。

### Step 2 — `student` シート（S1）を読む

1. `values` を上から順に見て、見出し行（「生徒氏名」「志望校」など）と値の組をリスト化する。  
2. 特に次のキーを確認する: `目標スコア`、`現状スコア`、`前回の試験結果`、`IA`（値の有無）、`EE トピック`、`HL` / `SL`、`英語力`、`次回試験日程`（空欄か）。  
3. **読み取り可否**: 値がセルにあるか／空かを記録する（空なら自動では埋められない）。

### Step 3 — 【SL】lesson plan（S2）の 4月分だけ抜き出す

1. 「Calculus」ブロック直上の **Probability `Unit test 2`** 行を探す（本データでは授業日 `2026/04/02`、配点 34・得点 34・率 100%）。  
2. `2026/04/13` に紐づく **Subtopic 一覧** を列挙する（Limits 〜 product rule まで quotient は別日）。  
3. `2026/04/30` に紐づく **quotient rule／指数・対数の微分行** と、メモ列の **「学校のTrigonometryのテストの結果確認と解説…」「2コマ分」** を転記する。  
4. 5月頭の `2026/05/04` 付近（trig derivative / Tangents / Normals）があるかだけ確認する（v2 の「今後」文と対応させる）。

### Step 4 — 教師MTGメモ（S3）を読む（日付ヘッダで会議単位を確認）

1. 先頭の日付 **`4月 30, 2026`** と担当表記 **`教師MTG (Runa Ninomiya)`** を確認する。  
2. **詳細パート**で、数値・固有名詞・動詞（確認した／予定／依頼）をマークする: 例）Trig テスト **7**、**5月4日** 仮日程、**6月22日** 修了見込み、**Section B**、**マークスキーム**、**IA/EE**、**ログ・指数・サインコサイン**、**Singapore 志望** など。  
3. **誤記注意**: メモ本文の「クラ氏」は **十倉さん家庭向けには置換済み**（管理者向け注記どおり）。比較は「情報の内容」が主で、固有名詞の typo は音声照合へ。

### Step 5 — HTML の該当箇所に貼り付けて一致判定する

各行について次のラベルを付ける。

| ラベル | 意味 |
|--------|------|
| **読取-A** | 一次ソースに**ほぼ原文レベル**で書いてある（機械転記しやすい） |
| **読取-B** | ソースはあるが**要約・推定・IB用語への翻訳**が必要 |
| **読取-C** | **セル／メモに無い**、または空欄（別取材・運用入力が必要） |
| **載せ方-D** | テンプレ方針で**あえて家庭向けに載せない**・別セクションへ送った |

**反映**列:

- **反映済** — ソースの情報が HTML に明示的にある（言い換えは可）。  
- **一部** — 一部だけ転記、ニュアンス差や欠落がある。  
- **未反映** — ソースにあり HTML に無い。  
- **意図非掲載** — 読めるがレイアウト／スキルで載せない方針。  
- **要照合** — ソース記述とレポート文言が論理的に食い違う可能性（要人手確認）。

### Step 6 — 「手動で過去込み」の典型を切り分ける

- **読取-A かつ 未反映** → 自動生成バグまたは未実装フィールドとして優先度高。  
- **読取-B かつ 反映済**でも、家庭向け文言に直した結果ユーザーが編集している → **正しい手動**。  
- **読取-C** を HTML が埋めている → **口述・チャット・過去ドラフト依存**の手動だった可能性が高い。
