# 月次レポート（HTML）を PDF にせずセキュアに送る

**目的**: 体裁は **HTML のまま**保ちつつ、保護者・第三者への誤開示やリンク流出を抑える。  
**正本のセキュリティ方針（システム化時）**: [T1-3 セキュリティ要件定義書](../../project/T1-3_セキュリティ要件定義書.md)（**署名付き URL ＋サーバ側でトークン検証**。本文は URL だけ送り、HTML は API 経由で返す想定）

---

## 1. 推奨（本番・個人情報を守る）

| 方式 | 内容 |
|------|------|
| **T1-3 型の配布** | メールには **推測不能なトークン付き HTTPS URL** のみ。閲覧時にサーバがトークンを検証し、**HTML を都度返却**（または短期キャッシュ）。PDF 化は不要。 |
| **HTTPS** | 送信リンク・閲覧は必ず TLS。HTTP 直リンクは避ける。 |

※ ブラウザ単体で「この HTML ファイルにパスワードをかける」標準機能はありません。パスワードだけでファイルを守るなら **ZIP 等のアーカイブ**（下記 2）か、**クラウドのリンク権限**（下記 3）が現実的です。

---

## 2. 手動でもすぐできる（HTML ファイルをそのまま添付・共有）

| 方式 | メリット | 注意 |
|------|----------|------|
| **パスワード付き ZIP** | HTML を変換せず同梱できる。メール添付で送れる。 | パスワードは **別チャネル**（電話・口頭・別メール）で伝える。ZIP は **AES-256**（7-Zip / OS 標準ツールの該当機能）を推奨。 |
| **Google Drive / OneDrive / Box** | リンクの閲覧者を **組織内限定** や **特定ユーザー** にできる。 | 「リンクを知っている全員」は弱い。共有範囲を必ず確認。 |

---

## 3. 避けたいこと

- **推測しやすい URL**（生徒番号・氏名がパスに入る等）だけで公開する。
- **平文メールに HTML 全文を貼る**（転送・ログ流出でそのまま漏れる）。
- **クライアントだけで「暗号化 HTML」**（パスワードがソースやブラウザに残りやすく、本番の保護には不向き）。

---

## 4. 社内ツール（全文エディタ）との関係

[tools/monthly_report_full_editor.html](tools/monthly_report_full_editor.html) で編集したあと、**正本**はリポジトリの `docs/samples/monthly-reports/` 側に反映し、保護者向けには上記 **1 または 2** の経路で送る運用にすると、PDF なしでも方針と整合します。

**エディタの「送付エクスポート」**（PBKDF2 + AES-GCM で **1 枚の HTML** に包み、送付用パスワード・URL を表示）は、手早く共有を区切りたいときの**補助**です。暗号化済みでも「ファイルを転送した相手が離脱できない」わけではなく、[§3](#3-避けたいこと) の注意と **[T1-3](../../project/T1-3_セキュリティ要件定義書.md) 型の配布**のどちらを正とするかは運用で切り分けてください。
