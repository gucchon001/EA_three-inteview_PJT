うまく動かないとき
  • 送付用 URL が空・HTTP 501：python -m http.server では POST がなく自動発行できません。必ず python scripts/serve_project.py で起動し、表示されたポートの URL からエディタを開き直してください。
  • サンプルが読めない：file:// では fetch できません。python scripts/serve_project.py で起動した http://127.0.0.1:… から開くか、ファイルを開くで HTML を指定してください。
  • 表示が古い／真っ白：ブラウザ下書きが優先されています。下書き削除または高藤でリセットを試してください。
  • 正本とエディタの整合：サンプル（高藤・鈴木）の URL は ../reports-manifest.json に一元化しています。正本 HTML を更新した人はマニフェストの revision も更新してください。下書き保持中に rev が変わると、起動時に緑メッセージで知らせます。
  • 送付エクスポート：暗号化 HTML 保存後、送付用 URL を自動発行します（同一サイトへ POST /api/monthly-protected-upload）。ローカルは serve_project.py がディスクの tools/_published/ に保存。Vercel は Blob ストア連携で BLOB_READ_WRITE_TOKEN が自動付与されれば追加設定なしで動作します（未連携時は相対パス目安のみ）。パスワードは別経路で伝えてください。詳細は SECURE_DELIVERY。
  • 送付 API の上書き：別ドメインの受け口を使う場合のみ、localStorage.setItem('ea_monthly_protected_upload_url', 'https://…') または meta name="ea-monthly-protected-upload" を設定してください（応答は {"ok":true,"url":"…"}）。
  • 別ブラウザ・他PCで続ける：下書き（ブラウザ内の自動保存）は共有されません。ファイルに保存で HTML を書き出し、続きからはファイルから開くで同じファイルを選んでください。
  • 絵文字：ツールバーの「絵文字挿入」で枠内のカーソル位置に入れられます（HTML を直接編集のソースにフォーカスがあるときはそちらに挿入）。修正は該当箇所を選択してからキーボードで打ち直しても構いません。

送付用 URL

http(s) では完了時に /api/monthly-protected-upload へ送り、宛先がブラウザで開く公開 URL を入れます（ページ表示用。ファイル専用のダウンロード URL ではありません）。開くと先にパスワード入力が出て、正しいとレポート本文が表示されます。失敗時は欄を空にし、画面下の緑メッセージに手順が出ます。URL に ?download=1 を付けないでください（保存ダイアログになります）。

自動発行に成功したときだけ URL が入ります。手動で配るときは、ダウンロードしたファイルを docs/samples/monthly-reports/tools/ に置いたうえで、その http URL を入力してください。
