## 検出アルゴリズム（仕様）

```
for each block (Block 1..7):
    label = アンカー文字列
    expected_col = 期待列
    found = scan column[expected_col±1] for label, in row range [min, max]
    if not found:
        record import_error("missing block", block_name)
        continue
    block.start_row = found_row
    block.data_rows = next N rows until next block's anchor or blank line

科目数 N の判定:
  Block 2 のヘッダ行 +1 から、`No` 列が 1/2/3 で連続する行数を数える。
  最大 3 まで（4以上は import_error）。
```

---

## 例外パターン

- **A型タブ**: Block 7 が存在しないか、行数が浅い → 月別記録は欠損として処理
- **空欄率の高いタブ**: テンプレ未編集に近い（近添・伴場）→ Block 1, 2 のみ抽出して Block 3-6 はスキップ可
- **マージセル**: ラベルがマージ範囲の左上にしかないため、`merged_ranges` を参照して論理的に同じセルとして扱う
- **左端列 (col A)**: 生徒一覧へのリンクや「生徒カルテ」のような独自ラベルが入る場合あり → パーサーは無視

## R-2 既知の制約: チェックボックスのデフォルト FALSE

指導報告書のチェックボックス（D列以降の月次セル）は、**未操作と明示 FALSE が区別できない**。
パーサーは値そのままを保存し、「報告書提出済み」判定は別シグナル（実施時間 > 0 など）と組み合わせる。
