## 主要な決定事項

### 1. 教師ミーティングのルール改善
**決定**: 教師に固有名詞（単元名など）を明確に発言してもらうように依頼する。
- **目的**: 文字起こしの抽象性を減らし、後工程の修正時間を削減
- **背景**: 現在、AI文字起こしで「ここの部分」「あれ」といった指示語が多用され、修正に時間がかかる

### 2. レポートの基本方針
**決定**: レポートは単発の情報ではなく、時系列での変化（線）を重視した内容とする。
- **目的**: 前回の状況と比較してどう変化したかという「線」の情報を提供
- **方法**: 時系列の変化を可視化（グラフ・表を使用）

### 3. 文字起こしファイルの命名規則
**決定**: 自動取り込みのため、文字起こしファイルのタイトルに「生徒番号」と「教師番号」を含める。
- **目的**: システムが自動で検知・取り込む仕組みの実現
- **適用**: Google Driveに保存されるレコーディングファイル

## 開発中のカリキュラム管理システム（現状）

### 進捗状況
- **現在の状態**: Speaker 2のPCでのみ動作する状態
- **共有計画**: 明日には共有し、微修正を進められるようにする予定

### 実装済み機能
1. **面談メモ入力機能**
   - Googleドキュメントからの入力、またはシステムへの直接入力が可能
   - 認証後、自動的にテーブル形式で反映
   - 表示形式: テーブル表示（スプレッドシート風）とレコード表示を切り替え可能
   - 特定の人物で絞り込み表示が可能

2. **学習計画表作成機能**
   - A項からE項までのデータを入力後、「学校シラバスベース」か「IBOベース」かを選択して計画表を生成
   - IBOのシラバスはシステムに読み込み済み
   - 学校のシラバスを追加で入力すると、両者のマッピングが視覚的に表示
   - マッピング画面で内容を修正後、最終的な学習計画表として出力

3. **AIを活用したスケジュール自動生成機能**
   - 備考欄に「苦手」などのキーワードを入力すると、AIが解釈
   - 苦手な項目は2回分、それ以外は1回分といった形でスケジュールを自動生成
   - 目標: AIがインプットを基にベースとなる提案を全自動で行えるようにする

### 今後の開発方針
1. **優先機能**: 学習計画表などをスプレッドシートに出力する機能
2. **管理画面**: システムの運用が軌道に乗れば、教師と共有するための管理画面を構築
3. **マイページ**: 将来的に家庭向けのマイページ機能として展開

## レポート自動化システムの要件

### インプット情報
1. **各種ミーティングの文字起こし**
   - 教師ミーティング
   - 保護者面談
   - 三者面談

2. **学習計画表**
   - 現在の進捗状況
   - 目標設定

3. **将来的に追加**
   - 指導内容そのものの文字起こし

### レポート構成（提案案）
```
【EDUBAL Academy 月次学習レポート】
生徒名：〇〇 様 ／ 対象期間：〇月
担当教師：〇〇 ／ 担当CA：〇〇

──────────────────────────
Section 1｜今月の学習成果（肯定的フィードバック）
  - 修得した単元
  - 正答率の推移（グラフ or 数値）
  - ひとこと評価

Section 2｜現状の課題と分析（逆算データ）
  - 目標スコア vs 現状スコア
  - 試験まで残り〇週 ／ 必要授業回数 〇回 ／ 現状ペースだと〇回不足
  - ボトルネック単元

Section 3｜今後の学習計画案
  - A案（現状維持）：リスクと必要な自習量
  - B案（推奨）：増回 or 延長 ／ 月額〇〇円
  - C案（長期休暇集中）：冬/春休みの活用

Section 4｜ご家庭へのお願い
  - 情報提供のお願い（シラバス・答案）
  - 三者面談のご案内・候補日
──────────────────────────
```

### 自動化フローのイメージ
```
文字起こし → 報告書自動作成 → 報酬自動送信 → 請求書送付
```

## 業務フローの理想形

### 現在の課題
- **教師ミーティング**: 1時間/件
- **メール作成時間**: 4〜6時間/件（藤井さん属人）
- **品質問題**: AI要約の品質が低く、大幅修正が必要

### 目標状態
- **教師ミーティング**: 30分/件
- **レポート作成時間**: 30分/件
- **自動化レベル**: AIでレポート自動生成、人間は微修正のみ

### 理想フロー
```
【事前】診断サービスで情報収集（答案・模試・スケジュール）
  └→ 教師 MTG（@0.5h）── アジェンダ標準化
       └→ 文字起こし → AI でレポート自動生成（@0.5h）
            └→ 家庭面談（@1h）── 面談メモビューで情報参照
                 └→ 面談結果をレポートに統合
                      └→ 保護者向けレポート（PDF）送付
```

## 懸念事項・課題

### 技術面
1. **データインポートの遅延**: 「タイルメーカー」が重く、データ読み込みだけで2時間近くを要する
2. **文字起こし精度**: 教師の発話が抽象的で、AIが正確に解釈できない
3. **セキュリティ**: 個人情報を含むレポートの安全な配布方法

### 運用面
1. **教師の協力度**: 固有名詞を明確に発言してもらうための教育と動機付け
2. **ファイル管理**: 命名規則の徹底と遵守
3. **権限管理**: 複数メンバーでのGoogle Drive共有設定

### 品質面
1. **AI要約の品質**: 過剰な表現や抽象的な表現の解読が必要
2. **標準化**: 教師の裁量に依存している部分を標準化する方法
3. **可視化**: 時系列での変化を効果的に伝える表現方法
