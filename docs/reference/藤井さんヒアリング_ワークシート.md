### 提案：月次レポート セクション構成

```
【EDUBAL Academy 月次学習レポート】
生徒名：〇〇 様  ／  対象期間：〇月
担当教師：〇〇  ／  担当CA：〇〇

──────────────────────────
Section 1｜今月の学習成果（肯定的フィードバック）
  - 修得した単元
  - 正答率の推移（グラフ or 数値）
  - ひとこと評価

Section 2｜現状の課題と分析（逆算データ）
  - 目標スコア vs 現状スコア
  - 試験まで残り〇週 ／ 必要授業回数 〇回 ／ 現状ペースだと〇回不足
  - ボトルネック単元

Section 3｜今後の学習計画案
  - A案（現状維持）：リスクと必要な自習量
  - B案（推奨）：増回 or 延長 ／ 月額〇〇円
  - C案（長期休暇集中）：冬/春休みの活用

Section 4｜ご家庭へのお願い
  - 情報提供のお願い（シラバス・答案）
  - 三者面談のご案内・候補日
──────────────────────────
```

### 週次会議での決定事項

#### レポート形式
- **基本方針**: 単発の情報ではなく、時系列での変化（線）を重視
- **推奨形式**: HTML形式（認証コード付き）を検討
  - PC/モバイル表示の最適化
  - セキュリティ対策として認証コードを入力すると表示
  - 過去レポートの一覧表示が可能
- **代替案**: PDF形式（A4縦長、モバイル対応）

#### 視覚化のポイント
- 時系列の変化をグラフ・表で可視化
- 「苦手分野が得意分野に変わった」「正答率が上がった」などの進捗を明確に
- 図やグラフを挿入してストレスなく情報伝達

### メール vs レポートの差分（藤井さんと確認）

| 項目 | 現行メール | レポート（目標） |
|-----|----------|---------------|
| 媒体 | メール本文（テキスト） | HTML形式（認証コード付き） |
| レイアウト | 文章のみ | セクション分け ＋ 表・数値・グラフ |
| 正答率 | 文中に埋め込み | 数値欄に明記 ＋ 時系列グラフ |
| A/B/C案 | 文章で説明 | 3案を対比表で提示 |
| 料金 | 文中に記載 | 明示欄あり |
| CTA | 文末に面談案内 | ボタン or 返信フォームへの誘導 |
| 時系列データ | ほぼなし | 前月比・変化の可視化を重視 |

## STEP 4：藤井さんの修正パターン（品質基準の言語化）

現在AIが生成した内容を藤井さんが大幅修正している。その修正パターンを確認し、AI生成の品質基準として言語化する。

### 週次会議で特定された問題点

1. **過剰な表現**
   - AI出力: 「スコア7は素晴らしい結果です」→「生徒様は7という素晴らしい結果を残し、とても良い成績を収められました」
   - 問題: 単純な内容を過度に丁寧な表現で冗長化

2. **抽象的な表現の解読**
   - 教師発言: 「ここの部分」「あれ」といった指示語
   - 問題: 文字起こしされた際に具体的な内容が不明になり、修正に時間がかかる

3. **時系列データの不足**
   - 問題: 単発の「点」の情報のみで、前回からの変化が分からない

### 確認項目

**文章のトーン・文体**
- [ ] どんな言葉遣いに直すことが多いか（丁寧すぎる？固すぎる？）
- [ ] 保護者に「響く」フレーズと「冷たく聞こえる」フレーズの違いは？
- [ ] 過剰な表現を簡潔にする基準は？

**数値の扱い**
- [ ] AIが出した数値をどのくらいの頻度で修正するか
- [ ] 数値の根拠が弱い場合、どう補うか
- [ ] 時系列データ（前月比）の扱いはどうするべきか

**提案の論理構成**
- [ ] AI案が「弱い」と感じるのはどんな時か
- [ ] 「これは送れない」と判断する基準は何か
- [ ] 4トリガーと提案の結びつきが明確か

**構成・順序**
- [ ] 現行メールの順番でよいか、変えたい部分はあるか
- [ ] 「計画」の話を「肯定的フィードバック」の後に入れるべきか

### 品質チェックリスト（ヒアリング後に完成させる）

| チェック項目 | 基準 | OKの例 | NGの例 |
|------------|------|-------|-------|
| 肯定フィードバックの具体性 | 単元名 ＋ 数値が入っているか | 「Functions の正答率が前月比15%向上」 | 「頑張っています」 |
| 逆算データの明示 | 残り週数・不足回数が数値で入っているか | 「残り8週で5回分不足」 | 「このままでは難しい」 |
| 提案の論理的必然性 | 弱点 → 提案の因果関係が明確か | 「正答率60% → 目標70%に達するには追加演習が必要」 | 「もっと勉強しましょう」 |
| 料金の透明性 | B案以降に月額・総額が明記されているか | 「月額+5,000円（週1回追加）」 | 「少し追加費用がかかります」 |
| CTAの具体性 | 面談候補日が3つ以上提示されているか | 「4/10(木)18:00、4/12(土)10:00、4/13(日)14:00」 | 「ご都合の良い日時をお知らせください」 |
| 固有名詞の使用 | 抽象的な指示語ではなく具体的な単元名・問題番号 | 「微分法の応用問題No.3」 | 「ここの部分」 |
| 時系列変化の記載 | 前回からの変化が明確か | 「前月比+10ポイント向上」 | 「良くなっています」 |

## STEP 5：システム連携と自動化

### 自動取り込みシステム
- **目的**: 文字起こしファイルをシステムが自動で検知・取り込む
- **命名規則**: `生徒番号_教師番号_面談種別_日付.ext`
  - 例: `STU001_TCH003_教師MTG_20260402.txt`
- **アクション**: @[Speaker 2] が本山さん・藤井さんのレコーディングフォルダへのアクセス権限をリクエスト

### レポート生成フロー
```
文字起こしファイル自動取り込み
  → データパース（生徒番号・教師番号・内容）
  → AIによるレポート自動生成
  → 藤井さんがマイページ上で編集
  → HTML形式でレポート生成
  → メール送信画面自動生成（テンプレート＋URL挿入）
  → ボタン一つで送信完了
```

### インプット情報の確定
1. **各種ミーティングの文字起こし**
   - 教師ミーティング
   - 保護者面談
   - 三者面談
2. **学習計画表**
   - 現在の進捗状況
   - 目標設定
3. **将来的に追加**
   - 指導内容そのものの文字起こし
